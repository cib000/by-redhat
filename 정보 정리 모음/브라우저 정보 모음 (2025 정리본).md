# 브라우저 정보 모음 (2025 정리본)


---

윈도우에서 사용 가능한 주요 웹 브라우저는 다음과 같습니다.



마이크로소프트 엣지 (Microsoft Edge)

윈도우에 기본 설치된 브라우저로, 크로미움 엔진 기반. 

빠른 성능과 마이크로소프트 서비스 통합이 특징.




구글 크롬 (Google Chrome)

가장 널리 사용되는 브라우저 중 하나로, 

크로미움 엔진 기반. 빠른 속도, 다양한 확장 프로그램, 

구글 서비스와의 높은 호환성.





모질라 파이어폭스 (Mozilla Firefox)

오픈소스 브라우저로, 개인정보 보호와 사용자 맞춤 설정에 강점. 

독립적인 렌더링 엔진(게코) 사용.





오페라 (Opera)

크로미움 기반 브라우저로, 내장 VPN, 광고 차단, 

메신저 통합 등의 독특한 기능 제공.






브레이브 (Brave)

크로미움 기반, 개인정보 보호와 광고 차단에 초점을 맞춘 브라우저. 

빠른 속도와 암호화폐 보상 시스템이 특징.






토르 브라우저 (Tor Browser)

익명성과 개인정보 보호를 위한 브라우저. 

파이어폭스 기반으로, 토르 네트워크를 통해 데이터 암호화.





주요 웹 브라우저

마이크로소프트 엣지 (Microsoft Edge)

크로미움 엔진 기반, 윈도우 기본 브라우저, 마이크로소프트 서비스 통합.





구글 크롬 (Google Chrome)

크로미움 엔진 기반, 빠른 성능, 다양한 확장 프로그램, 구글 서비스 통합.




모질라 파이어폭스 (Mozilla Firefox)

게코 엔진 기반, 오픈소스, 개인정보 보호 및 커스터마이징 강점.



오페라 (Opera)

크로미움 엔진 기반, 내장 VPN, 광고 차단, 메신저 통합.




브레이브 (Brave)

크로미움 엔진 기반, 개인정보 보호, 광고 차단, 암호화폐 보상 시스템.




토르 브라우저 (Tor Browser)

파이어폭스 기반, 토르 네트워크를 통한 익명 브라우징.





## 기타 주목할 만한 브라우저



비발디 (Vivaldi)

크로미움 엔진 기반, 고급 사용자 맞춤 설정, 탭 관리 기능.




UC 브라우저 (UC Browser)

크로미움 기반, 데이터 압축 및 빠른 다운로드 기능.




맥스톤 (Maxthon)

클라우드 동기화와 경량화에 초점, 크로미움 및 웹킷 엔진 사용.




슬림젯 (Slimjet)

크로미움 기반, 속도 최적화와 추가 기능(예: 비디오 다운로더) 제공.



에픽 프라이버시 브라우저 (Epic Privacy Browser)

크로미움 기반, 개인정보 보호에 특화, 추적 방지 및 VPN 내장.




코모도 드래곤 (Comodo Dragon)

크로미움 기반, 보안 및 개인정보 보호 강화.




시몽키 (SeaMonkey)

모질라 기반, 브라우저와 이메일 클라이언트, HTML 편집기 통합.




팔문 브라우저 (Pale Moon)

파이어폭스 포크, 가벼운 성능과 클래식 인터페이스 제공.




워터폭스 (Waterfox)

파이어폭스 기반, 64비트 시스템에 최적화, 개인정보 보호 강화.




넷서프 (NetSurf)

경량 오픈소스 브라우저, 저사양 시스템에 적합.




K-Meleon

파이어폭스 기반, 경량화된 브라우저, 저사양 PC에 적합.




아반트 브라우저 (Avant Browser)

웹킷 및 트라이던트 엔진 기반, 빠른 렌더링과 다운로드 관리.





SRWare Iron

크로미움 기반, 구글의 추적 기능 제거에 초점.





QTWeb

웹킷 기반, 경량화된 포터블 브라우저.

# 참고

크로미움 엔진은 오픈소스 프로젝트로, 

위 브라우저들은 이를 기반으로 각자의 기능과 인터페이스를 추가하여 차별화.

크로미움 기반 브라우저는 일반적으로 크롬 웹 스토어의 확장 프로그램과 호환 가능.

목록에 포함되지 않은 마이너 브라우저 중에도 크로미움 기반인 경우가 있을 수 있으나, 

위는 윈도우에서 일반적으로 사용되는 주요 브라우저들입니다.






### 윈도우 운영 체제에서 크로미움 기반 브라우저(예: 구글 크롬, 마이크로소프트 엣지 등)가 실행될 때

 **윈도우 API에서 사용하는 윈도우 클래스 이름(Window Class Name)**을 의미하는 것으로 보입니다. 

 이 이름은 윈도우의 창 관리 시스템에서 특정 애플리케이션 창을 식별하기 위해 사용됩니다. 

 크로미움 엔진 기반 브라우저는 대부분 창을 생성할 때 

 Chrome_WidgetWin_1이라는 윈도우 클래스 이름을 등록하여 사용합니다.
 




 
아래는 이전에 언급한 크로미움 기반 브라우저들이 사용하는 윈도우 클래스 이름과 관련된 정보입니다. 

대부분의 크로미움 기반 브라우저는 Chrome_WidgetWin_1을 사용하지만, 

일부는 변형된 이름을 사용할 수 있습니다.





### 크로미움 기반 브라우저와 윈도우 클래스 이름

윈도우 클래스 이름은 브라우저가 운영 체제와 상호작용할 때 

창(Window)의 속성을 정의하는 데 사용됩니다. 

아래는 크로미움 기반 브라우저와 

그들이 사용하는 윈도우 클래스 이름에 대한 분석입니다:





## 구글 크롬 (Google Chrome)

윈도우 클래스 이름: Chrome_WidgetWin_1

설명: 크로미움 엔진의 표준 윈도우 클래스 이름. 

크롬의 기본 창(탭, 툴바, 콘텐츠 영역 포함)은 이 클래스를 사용.





## 마이크로소프트 엣지 (Microsoft Edge)

윈도우 클래스 이름: Chrome_WidgetWin_1

설명: 엣지는 크로미움 기반으로 전환된 이후 크롬과 동일한 윈도우 클래스 이름을 사용. 

일부 엣지 고유 기능(예: 웹 위젯)에서 변형된 이름이 사용될 수 있음.






## 오페라 (Opera)

윈도우 클래스 이름: Chrome_WidgetWin_1

설명: 오페라도 크로미움 기반이므로 기본적으로 Chrome_WidgetWin_1을 사용. 

사이드바나 팝업 창은 추가 클래스 이름을 가질 수 있음.






## 브레이브 (Brave)

윈도우 클래스 이름: Chrome_WidgetWin_1

설명: 크로미움 기반으로, 표준 Chrome_WidgetWin_1 사용. 

브레이브의 프라이버시 기능은 UI에 영향을 주지 않으므로 클래스 이름은 동일.






## 비발디 (Vivaldi)

윈도우 클래스 이름: Chrome_WidgetWin_1

설명: 비발디는 고도로 커스터마이징된 UI를 제공하지만, 

기본 창은 크로미움의 Chrome_WidgetWin_1을 사용.






## UC 브라우저 (UC Browser)

윈도우 클래스 이름: Chrome_WidgetWin_1 (또는 변형 가능)

설명: UC 브라우저는 크로미움 기반이지만, 자체 UI 요소나 모바일 중심 설계로 인해 

일부 창에서 변형된 클래스 이름을 사용할 수 있음. 

기본 창은 대체로 Chrome_WidgetWin_1.






## 맥스톤 (Maxthon)

윈도우 클래스 이름: Chrome_WidgetWin_1 (또는 자체 클래스)

설명: 맥스톤은 크로미움과 웹킷 엔진을 혼용하므로, 크로미움 기반 창은 

Chrome_WidgetWin_1을 사용하지만, 이중 엔진 특성상 다른 클래스 이름이 사용될 수도 있음.





## 슬림젯 (Slimjet)

윈도우 클래스 이름: Chrome_WidgetWin_1

설명: 슬림젯은 크로미움 기반으로, 기본 창에 Chrome_WidgetWin_1을 사용.





## 에픽 프라이버시 브라우저 (Epic Privacy Browser)

윈도우 클래스 이름: Chrome_WidgetWin_1

설명: 크로미움 기반으로, 표준 Chrome_WidgetWin_1을 사용.






## 코모도 드래곤 (Comodo Dragon)

윈도우 클래스 이름: Chrome_WidgetWin_1

설명: 크로미움 기반으로, 기본적으로 Chrome_WidgetWin_1을 사용.


## SRWare Iron

윈도우 클래스 이름: Chrome_WidgetWin_1

설명: 크롬과 유사한 구조로, Chrome_WidgetWin_1을 사용.



## 추가 설명

Chrome_WidgetWin_1이란?:

이는 크로미움 프로젝트에서 사용하는 기본 윈도우 클래스 이름입니다. 

크로미움 기반 브라우저는 창을 생성할 때 윈도우 API의 RegisterClass 함수를 통해 

이 이름을 등록하여 창의 동작(예: 크기, 스타일, 메시지 처리)을 정의합니다. 

이 이름은 크로미움의 표준 UI 프레임워크와 관련이 있으며, 

대부분의 크로미움 기반 브라우저가 이를 그대로 사용합니다.





## 예외 및 변형:

일부 브라우저(예: UC 브라우저, 맥스톤)는 추가적인 UI 요소(팝업, 독립 창 등)에서 

자체적인 윈도우 클래스 이름을 사용할 수 있습니다. 

그러나 기본 브라우저 창은 대체로 Chrome_WidgetWin_1을 유지합니다.




확인 방법:

윈도우 클래스 이름을 확인하려면, 

윈도우에서 실행 중인 브라우저 창을 Spy++ 같은 도구나 

프로그래밍 방식(예: GetClassName API)으로 분석할 수 있습니다.




아래는 이전에 언급한 크로미움 기반 브라우저들의 공식 홈페이지 링크입니다.

엣지 내부 명령어 (edge://)
엣지 브라우저 주소창에 입력하면 특정 설정 페이지나 진단 도구로 이동합니다.

일반 설정 및 정보
명령어	           	설명
edge://settings		전체 설정 페이지
edge://extensions	확장 프로그램 관리
edge://downloads	다운로드 기록
edge://history		방문 기록
edge://favorites	즐겨찾기 목록
edge://version		브라우저 버전 및 경로 정보
edge://about		사용 가능한 내부 명령어 목록

시스템 및 성능 분석
명령어			설명
edge://gpu		GPU 상태 및 하드웨어 가속 정보
edge://sandbox		샌드박스 보안 상태
edge://tracing		성능 추적 도구
edge://histograms	성능 통계
edge://crashes		충돌 기록

네트워크 및 개발자 도구
명령어				설명
edge://net-export		네트워크 로그 저장
edge://webrtc-internals		WebRTC 통신 로그
edge://policy			조직 정책 확인 (기업용)
edge://device-log		디바이스 로그 (USB, Bluetooth 등)

실험적 기능 (edge://flags)
edge://flags를 입력하면 실험적 기능을 켜거나 끌 수 있는 페이지로 이동합니다.

예시:
#edge-tab-groups → 탭 그룹 기능
#enable-parallel-downloading → 병렬 다운로드
#smooth-scrolling → 부드러운 스크롤

# GitHub Desktop Private/Public 관리 체크리스트

| 구분          | 체크 항목 |
|:---------------:|------------------------------------------------|
| **Repository** | - Desktop 상단 Repository 이름 확인<br>- Private/Public 표시 확인<br>- Remote URL 확인 (올바른 GitHub 저장소 연결) |
| **커밋 전**   | - Changes 탭 확인 (변경 파일 목록 확인)<br>- 민감 정보/로그/환경 파일 포함 여부 확인<br>- Commit 메시지 스타일 확인<br>  - Public: 정제, 배포용<br>  - Private: 자유, 실험용 |
| **브랜치 관리** | - Private: 자유 브랜치 사용 가능<br>- Public: main/master 중심, 배포 브랜치만 사용<br>- Private → Public 이동 시 파일 선택적 복사 |
| **푸시 전**   | - Repository 확인 (Push할 저장소가 맞는지)<br>- Conflict 여부 확인<br>- 필요 시 Resolve (민감 파일 포함 주의) |
| **파일 관리** | - .gitignore 확인<br>  - Public: 로그, 환경 파일 제외<br>  - Private: 자유<br>- 폴더/파일 구조 확인 |
| **Backup & 안전** | - Private repo 정기 백업<br>- Public repo Release Tag 활용 가능<br>- Push 실수 대비 (force push 신중) |

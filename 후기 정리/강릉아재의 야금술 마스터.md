2️⃣ YAML 소개

YAML = YAML Ain’t Markup Language

가독성 높은 데이터 직렬화 형식

설정 파일, 자동화, DevOps 등에서 사용

2-1. 기본 문법

들여쓰기로 구조 표현

키-값 쌍: key: value

리스트: - 항목1

주석: # 설명

2-2. 예시
server:
  host: 127.0.0.1
  port: 8080
  ssl: true

users:
  - name: Alice
    role: admin
  - name: Bob
    role: user

2-3. Markdown과 연계

Markdown 내 YAML 블록 삽입 가능

```yaml

list:
  - item1
  - item2

- 단, 실제 프로그램 적용용으로는 **별도 `.yaml` 파일** 필요

## 3️⃣ 학습 전략: 야금야금 전술

> 하나씩 천천히, 완전히 내 것으로 만드는 전략

### 3-1. 과정
1. **이해**: 개념 파악
2. **실습**: 직접 코드 작성 / Markdown 작성
3. **기록**: 문서화 + GitHub 커밋
4. **반복**: 문제점 수정, 구조화, 체화

### 3-2. 장점
- 작은 단위라도 **완전히 습득** 가능  
- 문서화와 실습이 **결합되어 기억에 오래 남음**  
- 전체 그림 → 세부 기능까지 **체계적 정리 가능**

### 3-3. 팁
- 파일명 명확하게: `markdown_basics.md`, `yaml_intro.md`  
- 공통 README에서 전체 흐름 정리  
- 구분선, 헤더, 코드 블록, 목록, 이미지 활용으로 **가독성 극대화**  
- GitHub Push / Pull 시 경로, 인코딩 확인

# VS Code Markdown 이미지 문제 분석

## 1️⃣ 문제의 본질

- VS Code(Visual Studio)에서 Markdown 파일 저장 시 **ASCII 외 문자가 포함**되어 있다는 경고
- GitHub에서 Markdown 렌더링 시 **경로 인식이 깨질 수 있음**
- 특히 이미지 경로에 **한글, 공백, 특수문자**가 포함될 경우 문제 발생
- 파일명이나 경로에 **보이지 않는 비표준 문자 / BOM / 특수 공백**이 있을 수 있음
- Windows에서는 정상 표시 → GitHub에서는 깨짐 → 로컬에서 정상 표시되기 때문에 의문 발생

---

## 2️⃣ 가능성 있는 원인

### 1. UTF-8 BOM 문제
- VS Code가 파일을 UTF-8 BOM으로 저장 → GitHub Markdown에서 일부 경로 인식 문제 발생
- 경로/이미지 파일명에 BOM 문자 포함될 가능성 있음

### 2. 숨은 특수문자
- Markdown 내 보이지 않는 공백 문자, 탭, 한글 입력 시 **특수 유니코드 문자** 포함 가능
- 예: non-breaking space, `\u200B` (zero-width space)
- 이 경우 Markdown에서 경로나 구문 인식이 깨질 수 있음

### 3. 파일명/경로 문제
- 경로에 **한글, 공백, 특수문자** 포함 → GitHub에서는 `%20` 등으로 인코딩 처리되어 렌더링 문제 발생

---

## 3️⃣ 확인 / 해결 방법

### 1. 인코딩 변경
- VS Code: **파일 → 다른 이름으로 저장 → UTF-8 (without BOM)**
- 기존 파일을 이 방식으로 다시 저장 후 Push

### 2. 특수 문자 제거
- Markdown 파일에서 경고가 나타난 위치 확인
- 눈에 안 보이는 문자를 삭제하거나, 깨끗하게 다시 입력

### 3. 파일명과 경로 ASCII만 사용
- 이미지 파일명: 영어 + 숫자 + `-` 또는 `_` 만 사용
- 경로에도 한글/공백 피하기

### 4. GitHub Raw 링크 테스트
- 웹에서 Raw 열기 → 이미지 주소 확인
- 주소가 `%20` 등으로 깨져 있으면 **파일명/경로 문제**

---

## 💡 요약

- VS Code 경고 “아스키가 아닌 문자 포함”은 실제로 GitHub 이미지 렌더링 문제의 원인이 될 수 있음
- 특히 이미지 경로나 Markdown 파일 내부에 **숨은 BOM / 특수문자**가 포함되어 있는 경우,
  로컬에서는 정상 표시되지만 GitHub에서는 깨질 수 있음
